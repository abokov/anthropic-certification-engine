# 🚀 Anthropic GTM Technical Productivity Framework

This repository serves as a functional prototype for a **Technical Sales Onboarding Factory**. It is designed to solve the "Technical Fluency Gap" for global Go-To-Market (GTM) teams scaling in high-growth AI environments.

## 🎯 The Problem
As AI models like Claude evolve, GTM teams (Sales, Success, Applied AI) must move beyond high-level pitches to deep technical mastery of APIs, Tool Use, and Context Management. Manual onboarding and certification cannot scale at Anthropic’s velocity.

## 🛠️ The Solution: A Three-Tiered Enablement Loop

### 1. Tiered Curriculum (`/curriculum_labs`)
A hands-on, API-first lab series designed for different technical personas:
* **Account Executives:** Focus on reasoning and prompt safety.
* **Solutions Architects:** Focus on structured JSON output and system prompt engineering.
* **Customer Success:** Focus on token efficiency and TCO optimization.

### 2. Automated Validation (`/grading_engine`)
A Python-based **Certification Validator** that programmatically grades new hire submissions. By automating the feedback loop, we reduce "Time-to-Productivity" and remove the grading bottleneck from Subject Matter Experts (SMEs).

### 3. Continuous Improvement (`/analytics`)
The **Program Health Dashboard** tracks certification pass rates and identifies common friction points in the curriculum, allowing for data-driven iterations of the technical content.
